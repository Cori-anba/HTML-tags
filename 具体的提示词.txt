我要开发一个 HTML 标签速查手册网页（HTML Tag Reference Website），请你帮我完成这个项目。

项目目标：

制作一个用于快速查询 HTML 标签、标签属性、CSS 属性、全局属性的网页系统，方便我在写网页时快速查找标签的属性名、属性值、CSS样式及使用方法。重点是内容完整、查询方便，页面整洁、美观、实用即可。

---

【整体布局要求】

页面采用左右结构布局：

左侧：
固定可滚动的侧边栏（sidebar）

右侧：
主内容区域，即标签讲解页面（content）

页面顶部：
固定导航栏（header）

顶部导航栏只需要实现一个功能：

搜索框（支持搜索并跳转）

例如输入：

img
table
input
ul
class
id

回车即可快速跳转到对应标签的讲解页面。

---

【左侧侧边栏要求】

1、按照 HTML 标签英文首字母排序，列出所有常用 HTML 标签

例如：

a
div
form
img
input
li
meta
p
table
ul
video

等标签，列出的标签要包含当前版本的HTML所有支持的标签，当前版本的HTML语言已经遗弃的标签就不要列举

2、点击标签名称后：

右侧主内容区域跳转到对应标签讲解页面（可使用锚点定位）

3、当所有 HTML 标签列出完毕后：

侧边栏底部增加：

全局属性（Global Attributes）

点击后跳转到：

全局属性讲解页面

该页面需要详细讲解：

id
class
style
title
hidden
tabindex
data-*

等常用全局属性及其写法和注意事项

---

【每个标签讲解页面必须包含】

一、页面大标题

格式示例：

<ul>（unordered list：无序列表）
<aside> （aside n.侧边）
<div> （division n.分区）

---

二、功能和应用场景解释

用一小段话清晰说明、这段话不要分条

它属于单标签还是双标签（必须明确写出）
该标签的作用
常见的使用场景
（重点内容要使用 strong 标签进行强调）

例如：
ul（unordered list）标签为 <strong>双标签</strong>，它用于定义无序列表，即列表项（li）前默认带有圆点符号，专用于表示 <strong>一组并列、无特定顺序</strong> 的条目，典型场景包括导航菜单、功能列表、商品特征展示等。

要求：
内容要清晰、简洁、容易理解

---

三、属性表格（重点）

表格名称：

属性表格

共五列：

第一列：
只要不是那种几乎没有什么项目和个人使用的、极度不推荐的属性，统统写上去

要求：
* 按重要性和常用程度排序
* 单个属性可能占多行
* 属性名所在单元格要附带解释
* 即使是全局属性（id、class、style）也必须列出

第二列：
属性所对应的英文名以及词性和中文翻译

例如：
width属性对应：width n.宽度

第三列：
该属性所有属性值 / 写法，每个占一行（如果该属性包含的属性值过多，可以剔除掉极度不推荐的属性值或写法）

要求：
* 按常用程度排序，重点的要标星号
* 每个属性值占一行
* 内容过长允许换行
* 属性值需要附带解释

第四列：
属性值所对应的英文名

例如：

ltr → left to right（从左到右）

第五列：
注意和要点

要求：
说明该属性的重要注意事项

例如：
id 属性在同一个页面中必须唯一，不能重复

---

四、CSS 表格（重点）

表格名称：
CSS表格

要求：
该标签相关的所有常用 CSS 属性及属性值
表格结构与上一个模块的属性表格完全一致：

共五列：

CSS属性
英文原称
属性值
英文解释
注意事项

例如：

display
position
margin
padding
font-size
line-height
overflow
z-index

等

---

五、相关标签推荐

在页面底部增加相关标签推荐

例如：
ul 标签的页面大概会推荐：

ol
li
dl
dt
dd

要求：

使用超链接形式

点击后可直接跳转到对应标签页面

我用其他AI生成的该网站项目的项目结构如下，当然，如果你觉得你比其他AI聪明，你可以根据你自己的代码规范和风格进行相应的调整：
html-tag-reference/                     （项目总文件夹：整个 HTML 标签速查网站项目）

│
├── index.html                          （网站首页：整个项目的主入口页面）

│
├── css/                                （样式文件夹：专门存放 CSS 样式文件）

│   └── style.css                       （主样式文件：控制页面整体布局、颜色、表格、侧边栏、导航栏等样式）

│
├── js/                                 （JavaScript 文件夹：专门存放功能代码）

│   ├── main.js                         （主入口 JS：负责页面初始化、加载数据、启动整个网站）

│   │
│   ├── search.js                       （搜索功能：实现顶部搜索框输入后跳转到对应标签页面）

│   │
│   ├── render.js                       （内容渲染功能：根据 JSON 数据动态生成标签讲解页面内容）

│   │
│   └── sidebar.js                      （侧边栏功能：生成左侧标签列表、点击跳转、滚动定位等）

│
├── data/                               （数据文件夹：整个网站最重要的内容库）

│   ├── html-tags.json                  （HTML 标签数据：存放所有标签的信息，例如 div、img、ul、table 等）

│   │
│   ├── global-attributes.json          （全局属性数据：存放 id、class、style 等全局属性的详细说明）

│   │
│   └── css-properties.json             （CSS 属性数据：存放常用 CSS 属性及属性值说明）

│
├── components/                         （公共组件文件夹：可重复使用的页面部分）

│   ├── header.html                     （顶部导航栏组件：主要是搜索框区域）

│   │
│   └── sidebar.html                    （左侧侧边栏组件：标签列表区域）

│
├── assets/                             （资源文件夹：存放图片、图标等静态资源）

│   ├── icons/                          （图标文件夹：例如搜索图标、标签小图标等）

│   │
│   ├── images/                         （普通图片文件夹：用于页面展示）

│   │
│   └── screenshots/                    （截图文件夹：例如标签示例图、错误示例截图等）

│
├── pages/                              （页面模板文件夹：用于生成标签详情页面）

│   ├── tag-template.html               （标签讲解模板：所有 HTML 标签页面统一使用的模板）

│   │
│   └── global-template.html            （全局属性页面模板：专门用于全局属性讲解页面）

│
└── README.md                           （项目说明文档：记录项目介绍、使用方法、维护说明等）
